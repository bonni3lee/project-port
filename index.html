Approach

01

Build with Claude Code

My workflow runs on Claude Code: idea to working, interactive prototype in code, not static mocks. I test real interactions early and hand engineering something they can run.

02

Reference with Mobbin

I pull patterns and inspiration through Mobbin's MCP without leaving my workflow, so gathering references takes minutes instead of an afternoon of tab-hopping.

03

Prototypes live in GitHub

Every prototype is version-controlled in GitHub and co-owned with engineering and PM, so cross-functional partners can branch, review, and build on it directly.

04

Write skills, not one-offs

I write reusable Claude Code skills that fold my workflow into the team's, turning one-off speed-ups into something everyone can run.
